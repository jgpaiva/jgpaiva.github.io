Demo for delayed translation

This demo page shows a use-case where the client may only want to pay after X requests are made.

The server should generate an encrypted request which includes not only the hash of the text, but also the number of requests to be made before the data is translated by humans.

The user gets the generated page, presses the translate button and one of three things may happen:

  • The text was previously translated: the human translation is returned.
  • If the text was not previously translated, and the client threshold was reached, a machine translation is returned immediately, and the human translation is fetched on the background.
  • Otherwise: a machine translation is returned immediately, and the user is presented with a "crowd fund this translation?" button.
